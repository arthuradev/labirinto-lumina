# ROADMAP.md — Caminho até v1.0.0

Este roadmap define os marcos principais do projeto **Labirinto Lumina**.

O objetivo é chegar a uma versão `v1.0.0` completa, jogável, documentada, validada e publicada.

## Filosofia

Este projeto terá escopo fechado.

A meta não é adicionar infinitas features. A meta é entregar um jogo pequeno, original, completo e bem apresentado.

## Marcos

### v0.1.0 — Fundação documental e técnica inicial

Objetivo:

- criar documentação;
- criar metodologia IA-first;
- definir arquitetura;
- definir escopo;
- criar setup técnico mínimo;
- criar CI inicial;
- criar repositório público.

Status: etapa inicial.

### v0.2.0 — Núcleo técnico do jogo

Objetivo:

- criar estrutura de código;
- criar canvas funcional;
- criar game loop;
- criar estados básicos;
- renderizar tela inicial real.

### v0.3.0 — Movimento, labirinto e colisão

Objetivo:

- implementar jogador;
- implementar input;
- implementar movimento em grid;
- implementar colisão com paredes;
- criar primeira fase navegável.

### v0.4.0 — Coleta, pontuação e condições de fim

Objetivo:

- implementar fragmentos de luz;
- implementar nós de pulso;
- implementar pontuação;
- implementar vitória/derrota básica;
- adicionar testes de regras.

### v0.5.0 — Sentinelas

Objetivo:

- implementar inimigos originais;
- criar comportamentos simples;
- validar perseguição/patrulha;
- implementar modo instável durante pulso.

### v0.6.0 — Fases completas

Objetivo:

- criar três fases originais;
- balancear dificuldade;
- implementar progressão;
- documentar formato de fase.

### v0.7.0 — Telas, HUD e UX

Objetivo:

- tela inicial;
- tela de controles;
- pausa;
- game over;
- vitória;
- HUD;
- fluxo completo.

### v0.8.0 — Áudio, polimento e high score

Objetivo:

- áudio original/CC0;
- mute;
- efeitos visuais simples;
- high score local;
- polimento de apresentação.

### v0.9.0 — Release candidate

Objetivo:

- testes completos;
- CI forte;
- GitHub Pages;
- README final;
- revisão de originalidade;
- release candidate.

### v1.0.0 — Versão final

Objetivo:

- jogo completo;
- documentação final;
- release pública;
- GitHub Pages funcionando;
- checklist final aprovado.

## Depois da v1.0.0

Não planejar features futuras antes da v1.0.0.

Ideias futuras devem ir para uma seção separada, sem interferir no escopo fechado.
