# Checklist de release

Este checklist deve ser usado antes de criar releases relevantes.

## Antes de qualquer release

- [ ] `npm run check` passou.
- [ ] Build passou.
- [ ] Testes passaram.
- [ ] README está atualizado.
- [ ] CHANGELOG está atualizado.
- [ ] Documentação relevante está atualizada.
- [ ] Tag segue `vX.X.X`.
- [ ] Release notes foram escritas.

## Antes de v0.9.0

- [ ] Jogo completo está jogável.
- [ ] GitHub Pages funciona.
- [ ] Todas as fases são acessíveis.
- [ ] Não há erros graves no console.
- [ ] Originalidade foi revisada.
- [ ] Assets foram revisados.
- [ ] README tem link para jogar.

## Antes de v1.0.0

Produto:

- [ ] Tela inicial.
- [ ] Controles.
- [ ] 3 fases.
- [ ] Pontuação.
- [ ] Vidas.
- [ ] Sentinelas.
- [ ] Game over.
- [ ] Vitória.
- [ ] Pausa.
- [ ] High score local.
- [ ] Mute.

Qualidade:

- [ ] `npm run check`.
- [ ] Teste e2e.
- [ ] Build final.
- [ ] Sem erros no console.
- [ ] GitHub Actions passando.

Documentação:

- [ ] README final.
- [ ] User guide final.
- [ ] Controls final.
- [ ] Architecture atualizada.
- [ ] CHANGELOG completo.

Originalidade:

- [ ] Nenhum asset protegido.
- [ ] Nenhum nome problemático.
- [ ] Nenhum mapa copiado.
- [ ] Nenhum som copiado.
- [ ] Nenhuma referência confundível.

Release:

- [ ] Tag `v1.0.0`.
- [ ] Release notes finais.
- [ ] Link do GitHub Pages.
