# Metodologia de desenvolvimento com IA

## 1. Propósito

Labirinto Lumina é um experimento público de desenvolvimento de software com IA.

O projeto será criado integralmente por IA, usando GPT-5.5, com direção humana.

O objetivo é observar como uma IA se comporta quando trabalha em um projeto com:

- escopo fechado;
- documentação clara;
- arquitetura planejada;
- validações;
- commits organizados;
- versionamento;
- releases;
- publicação.

## 2. Papel da IA

A IA é responsável por:

- criar arquivos;
- escrever código;
- escrever documentação;
- implementar funcionalidades;
- criar testes;
- rodar validações;
- fazer commits;
- criar tags;
- preparar releases;
- manter consistência arquitetural.

## 3. Papel humano

A direção humana é responsável por:

- definir intenção criativa;
- aprovar troca de etapas;
- revisar entregas;
- decidir ajustes de escopo;
- avaliar resultado final;
- garantir que o experimento permaneça alinhado.

## 4. Regra de autonomia

A IA pode trabalhar de forma autônoma dentro da etapa atual.

Ao concluir uma etapa, deve parar e perguntar se pode avançar.

## 5. Documentos-base

A IA deve sempre consultar:

- `AGENTS.md`
- `SDD.md`
- `GSD.md`
- `ARCHITECTURE.md`
- `ROADMAP.md`
- `docs/legal/originality-policy.md`
- `docs/releases/version-plan.md`

## 6. Como evitar improviso

A IA deve:

- seguir a etapa atual;
- não implementar features futuras;
- não criar arquitetura paralela;
- não adicionar dependência sem motivo;
- não mudar decisões aceitas sem ADR;
- validar antes de commit;
- documentar mudanças relevantes.

## 7. Como medir sucesso

O experimento será considerado bem-sucedido se a IA conseguir entregar:

- jogo completo;
- código organizado;
- documentação clara;
- histórico de commits limpo;
- versões planejadas;
- release pública;
- GitHub Pages funcionando;
- identidade original;
- escopo fechado cumprido.
