# Checklist de revisão

Use este checklist ao final de cada etapa.

## Escopo

- [ ] A etapa atual foi respeitada.
- [ ] Nenhuma feature futura foi implementada.
- [ ] Nenhum arquivo não relacionado foi alterado sem motivo.
- [ ] O resultado corresponde ao `GSD.md`.

## Originalidade

- [ ] Nenhum nome protegido foi usado.
- [ ] Nenhum sprite protegido foi usado.
- [ ] Nenhum som protegido foi usado.
- [ ] Nenhum mapa conhecido foi copiado.
- [ ] A identidade visual permanece original.

## Arquitetura

- [ ] A lógica de jogo está separada da renderização.
- [ ] O Canvas não contaminou o núcleo.
- [ ] Sistemas têm responsabilidades claras.
- [ ] Nenhum arquivo virou um "deus".
- [ ] A documentação arquitetural continua verdadeira.

## Código

- [ ] TypeScript compila.
- [ ] Código está legível.
- [ ] Comentários explicam decisões importantes.
- [ ] Não há comentários óbvios demais.
- [ ] Não há duplicação desnecessária.

## Qualidade

- [ ] Lint passou.
- [ ] Format check passou.
- [ ] Testes passaram.
- [ ] Build passou.
- [ ] Não há erros relevantes no console.

## Git

- [ ] Commit está em português.
- [ ] Commit segue Conventional Commits.
- [ ] Push foi realizado.
- [ ] Tag foi criada apenas se aplicável.
- [ ] CHANGELOG foi atualizado quando necessário.
